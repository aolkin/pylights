AutoKey Functionality – Sending Data to EOL¶

EOL AutoKey’s method of putting its data into a format ETC consoles can understand relies on sending virtual keystrokes to Expression Off-Line. This is accomplished using the SendKeys functionality of pywinauto.application. Because of this, pywinauto is required to write data.

The sender Module¶

The sender module provides the functionality that actually sends keys to Expression Off-Line. pywinauto is initialized once, when the module is first loaded, at which point it will attempt to find an Expression Off-Line window. If it fails then, pywinauto.findwindows will throw a WindowNotFoundError, and the module will fail to load. Once the module has been loaded, there is no way to reinitialize it without reloading it completely.

Note

This module is not written very well at the moment, so its implementation is likely to change and improve.

The sender module provides one function, which is used internally by most of the rest of the software, especially entities.

sender.send(keys, speed=0.01, noalert=False)¶
Parameters:
  • keys – a SendKeys compatible keystring to send to EOL.
  • speed – How long to wait between virtual keypresses. 0.01 seconds usually works. See SendKeys for more information.
  • noalert – if True, disables the dialog boxes before and after sending keystrokes.

send() maximizes the back EOL window and gives focus to EOL before sending any keystrokes. It is important that it retains focus until all keystrokes have been sent, or unexpected results may occur. As long as noalert is False, the first time this function is called it will pop up a dialog box saying so.

Once all keystrokes have been sent, the back EOL window will be restored, and if noalert is False and it is the first time this function has been called, a dialog box will pop up to alert the user that it is done.

The keys Module¶

Technically the keys module is platform-independent, but it is not terribly useful except to provide keystroke sequences for send(), which is not, so it is marked as Windows-only.

The keys module’s central functionality is keys.names, its KeyMap, but it also provides some Higher Level Functionality that uses that KeyMap.

Module Internals¶

It is unlikely that you would need to create a new KeyMap, but it is still documented here because it provides a small API that may be useful occasionally.

class keys.KeyMap(dict)¶

A KeyMap is a dictionary mapping names to SendKeys-compatible keystrings or key sequences. It may map multiple names to the same key.

In addition to the dictionary interface provided, it defines three methods which should be used to add additional mappings:

addkeys(d)¶

This accepts a single argument, which should be a dictionary of the form:

{
    '<key(s)>': '<name>',
    '<key(s)>': ('<name1>','<name2>')
}

It will iterate over this dictionary, adding maps for each key/key sequence from the name or names specified. A real example of this can be seen in the Default Contents of keys.names.

addkey(key, *names)¶

This maps one or more names to a single key.

addseq(seq, *names)¶

This maps one or more names to a specially formatted sequence. Use this method when you want to map a key sequence that uses existing keys or sequences.

To include existing keys/sequences in your new sequence, simply enclose their names in < and >.

Examples from the keys module:

# Maps several names to pressing "enter" twice.
names.addseq("<enter><enter>","confirm","confirmed","dblenter","enterenter","doubleenter")

# Maps "forcestage" to a sequence that should put the console in stage mode no matter what.
names.addseq("<stage><macroenter><clear><stage>","forcestage")

# Maps "save" to the sequence needed to execute the "Write All to Disk" action.
names.addseq("<setup>3<enter>1<enter><enter><stage>","save")

# Maps "reset" to a sequence that will execute a system reset.
names.addseq("<setup>4<enter><s1><enter>","reset")

names.addseq("<blind><s6>","deletecue")
names.addseq("<blind><sub><s6>","deletesub","deletesubmaster")

# Maps "pause" to twenty presses of the space key, which is ignored by EOL.
names.addseq("<space>"*20,"pause")
keys.names¶

This KeyMap is the central dictionary mapping names to keys. It is used by the functions in this module to convert human-readable data to key sequences, and it also supports adding additional custom key sequences.

It is initialized with one or more names for every keyboard shortcut EOL supports, as well as the above addseq() examples. [For more information, see Default Contents of keys.names]

Because of the way the higher-level functions in the keys module work, all of the names in keys.names should be lowercase and without spaces. This is explained more in the function descriptions for get() and parse_str().

Higher Level Functionality¶

These functions provide higher-level access to keys.names.

keys.get(name, strict=False)¶

Looks up a name in names.

It will lowercase its argument before performing the lookup, and if it finds nothing it simply returns its argument as-is. As long as strict is False, it will also remove spaces from its argument. This allows easy lookups of button presses with multi-word names.

keys.list_to_str(keys)¶

Performs get() lookups on a list and joins the result together with spaces.

keys.parse_str(keys)¶

Accepts a single string argument. This should be a nice human-readable sequence of button presses, which will be converted into a string of key strokes suitable for sending to EOL.

It is extremely flexible, and can take almost anything you can throw at it. It will accept any capitalization and spacing, and will even catch multi-word button names. However, if a multi-word button name’s first word is itself a button name, it will treat it as that button.

parse_str() even allows you to add < and > around button names, in which case they will simply be ignored. However, as of now, there is no way to include comments in a keystring.

Examples:

>>> parse_str('Cue 7 Go')
'q 7 ^g'
>>> parse_str('Channel 1 at 50 '+
              'Channel 2 at 75 '+
              'Channel 5 at 27 '+
              'Record cue 7 \n Cue 7 <GO>')
'c 1 a 50 c 2 a 75 c 5 a 27 r q 7 q 7 ^g'
>>> parse_str('AB Go <stage> AB Clear')
'^g +{F1} ^a'

Default Contents of keys.names¶

The following code adds all of the keyboard shortcuts Expression Off-Line supports. Most keys have multiple names, for convenience, but all at least have the name of the physical button you would press to get the same functionality.

    '^a': 'abclear',
    '^g': ('abgo','go'),
    '^f': 'abhold',
    '^s': 'abrate',

    'n': 'and',
    'a': 'at',
    '^k': 'blackout',
    '+{F2}': ('blind','blindmode'),
    '^c': 'cdback',
    '^z': 'cdclear',
    '^b': 'cdgo',
    '^v': 'cdhold',
    '^x': 'cdrate',
    'c': ('channel','chan','ch'),
    '{DELETE}': 'clear',
    'q': 'cue',
    'd': ('dimmer','dim'),
    '{DOWN}': ('down','downarrow'),
    '{ENTER}': 'enter',
    'x': 'except',
    '+{F8}': 'expand',
    '+{F3}': 'fader',
    '!': ('focuspoint','fp'),
    'f': ('full','fl'),
    'g': 'group',
    '?': 'help',
    'b': 'label',
    '{F4}': 'learn',
    '{LEFT}': ('left','leftarrow'),
    'v': 'level',
    '{PGDN}': ('levelwheeldown','leveldown','wheeldown'),
    '{PGUP}': ('levelwheelup','levelup','wheelup'),
    'k': 'link',
    '#': ('loadsub','loadsubmaster'),
    'm': ('macro*','m*'),
    '^e': ('macroenter','menter'),
    '^w': ('macrowait','mwait'),
    'o': 'only',

    '+{F6}': 'park',
    'p': 'part',

The softkeys were added in a slightly different way, and can be access via "softkey#", "soft#", or "s#".

for i in range(1,9):
    names.addkey('^{{F{i}}}'.format(i=i),'soft{}'.format(i),
                 'softkey{}'.format(i),'s{}'.format(i))

In addition to these, the examples for addseq() are included.
