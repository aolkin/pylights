Built-In Entities¶

EOL AutoKey provides several common entities in submodules of the entities module.

Note

Unless otherwise specified, all documented modules on this page are submodules of the entities module.

All entities described on this page use The Entity Interface. As such, they all inherit send, which is used to send the entity to EOL.

Entity.send()

Sends the entity’s keystring to EOL through the sender module.

Cues¶

The cues module provides three cue classes and three classes used by some of the cue classes, as well as some useful constants for cue types.

The Cue Class – A Basic Cue¶

Of the five types of cues (Crossfade, Allfade, Blocking, Effect, and Subroutine), the first three should all be created using the Cue class. Those three cue types should then be differentiated by the cue TYPE field.

class entities.cues.Cue(num, **kwargs)¶

You must provide a cue’s number on instantiation, though it may be changed after.

This class is setup so that if a field is unspecified, it should use the console’s default.

A basic cue has the following fields:

    _fields = (CUE,TYPE,UPTIME,DOWNTIME,UPWAIT,DOWNWAIT,LINK,FOLLOW,RATE,LEVELS,LABEL)

Cue Type Constants¶

The cues module also provides three constants (with abbreviated aliases) for different cue types.

entities.cues.XF¶
entities.cues.CROSSFADE¶

The TYPE of a crossfade cue.

entities.cues.AF¶
entities.cues.ALLFADE¶

The TYPE of an allfade cue.

entities.cues.BF¶
entities.cues.BLOCKING¶

The TYPE of a blocking cue.

Subroutines – The SubroutineCue and SubroutineStep Classes¶

A Subroutine Cue should be not be created using the Cue class, but rather the SubroutineCue class.

Note

SubroutineCue does not inherit from Cue.

class entities.cues.SubroutineCue(num, steps=(), **kwargs)¶

A SubroutineCue functions much like an Cue, except with many fewer fields. Additionally, its TYPE will always be forced to 4.

    _fields = (CUE,STEPS,RATE,LABEL)

The STEPS field should never be set directly, instead, it is a fields.List to which SubroutineStep objects should be added.

class entities.cues.SubroutineStep(cue, type_=1, level=100, **kwargs)¶

A SubroutineStep has the following fields:

    _fields = (CUE,TYPE,LEVEL,UPTIME,DOWNTIME,FOLLOW)

Once again, the Cue Type Constants may be used for the TYPE field.

Effect Cues – The EffectCue and EffectStep Classes¶

Effect Cues should be created using the EffectCue class.

class entities.cues.EffectCue(num, steps=(), **kwargs)¶

An EffectCue has most of the fields of a Cue, but in most ways it functions more like a SubroutineCue. Its TYPE will be forced to 3.

    _fields = (CUE,STEPS,UPTIME,DOWNTIME,DWELL,LINK,FOLLOW,RATE,LABEL)

The STEPS field is a fields.List to which EffectStep objects should be added.

class entities.cues.EffectStep(channels={}, **kwargs)¶

An EffectStep has the following fields:

    _fields = (CHANNELS,TIME,UPTIME,DWELL,DOWNTIME,LOWLEVEL,HIGHLEVEL)

Its CHANNELS field is an EffectChannels object that inherits from and behaves as a fields.Levels object. After object creation, all changes should be made using that interface.

class entities.cues.EffectChannels¶

This class does little other than inherit from fields.Levels and redefine _channel_format.
